# 讲义：Vibe Coding 基础与实战 (60分钟)

**课程目标：** 帮助开发者从“盲目依赖AI的许愿者”蜕变为“掌控全局的AI指挥官”，掌握 Vibe Coding 的核心思维框架与实用工具。

---

## 1. 开场与破冰：什么是真正的 Vibe Coding？ (0-5分钟)

大家好，欢迎来到本节关于 Vibe Coding 基础的课程。

提到 Vibe Coding（直觉驱动编程/氛围编程），很多人第一反应是“用嘴写代码”——随便给个提示词，让 AI 一键生成。但今天我们要打破这个迷思。真正的 Vibe Coding 不是完全放弃控制权，而是**建立一种高带宽、低摩擦的人机协作工作流**。

为了达到这个状态，我们首先要改造我们的工具环境。

---

## 2. 环境准备：掌控你的上下文 (5-10分钟)

**核心概念：** 在让 Agent 为你工作之前，你必须知道自己身在何处。

### 案例：为 Claude Code 添加 Statusline
很多同学上来就用 `claude code` 一键直出项目。但跑着跑着就迷失了：我在哪个分支？我的工作路径是对的吗？最致命的是，当前会话的 Context Window（上下文窗口）还剩多少？

**实操演示：**
我们可以通过一段简单的配置，为 Claude Code 命令行底部添加一个 **Statusline（状态栏）**。
* **Git分支：** 明确当前修改的归属，防止污染主分支。
* **工作路径：** 确保 Agent 正在读取和修改正确目录下的文件。
* **剩余 Context Window：** 这是一个重要的“续航指标”。当上下文即将耗尽时，AI 会开始遗忘或产生幻觉，这时我们就需要进行记忆压缩或切分任务。

环境配置好后，我们来看看人机沟通的本质。

---

## 3. 核心心法：显化你的需求 (10-20分钟)

指挥 Agent 最大的坑是什么？是“隐晦”。AI 是字面意思的执行者，如果你让它去猜，它一定会给你个“惊喜（惊吓）”。

### 一个通俗的比喻：年夜饭桌上的二舅
想象一下过年回家，二舅在饭桌上阴阳怪气你：“哎呦，小张现在是大人物了，在大城市赚大钱，我们这帮穷亲戚高攀不起了。”

面对这种话，普通的应对是：
* **解释/否认：** “不不不，二舅怎么会呢，我就是个普通打工人。” —— 这样只会顺着他的语境，让他觉得拿捏了你，继续嘚瑟。

**高级的应对（显化翻译）：**
不要顺着他的话往下讲，你要**把他的潜台词翻译成大白话，直接砸在桌面上**：
* “二舅，您这话的意思是，您觉得我现在混得好了，忘本了，看不起您老人家了？”
这时候，二舅往往会连连摆手：“哎呀我就是随口开个玩笑，你怎么还当真了呢？”

### 把这套逻辑用在指挥 Agent 上
人类的语言充满上下文、潜台词和模糊地带，而指挥 Agent，就是要**把隐晦的语言，翻译显化成极其直接、毫无歧义的指令**。

不要对 AI 说：“帮我把这个页面弄好看一点，像苹果官网那种感觉。”
你要显化它：“使用 Tailwind CSS，背景色设为 `#F5F5F7`，字体使用 San Francisco，组件间距保持统一的 `rem` 比例，去掉所有多余的边框线。”

## 4. 主奴辩证法与危险的“一键直出” (20-30分钟)

既然 AI 这么强大，我们能不能干脆什么都不管了？

### Demo 演示：BTC 价格监控器
我们来做一个 Demo：写一个实时监控比特币价格的脚本。
使用命令：`claude code --dangerously-skip-permissions`
（*注：这是一个高危操作，意味着赋予了 AI 绕过权限直接执行代码的能力。*）

不到一分钟，代码写好了，跑通了。很爽对吧？但这背后隐藏着巨大的危机。

### 哲学思考：黑格尔的“主奴辩证法”
在黑格尔的哲学中，主人通过奴隶的劳动来满足自己的需求。一开始，主人是自由的，奴隶是被剥削的。但随着时间推移，奴隶在劳动中掌握了改造世界的技术和知识，变得越来越强大；而主人因为把一切都外包给了奴隶，丧失了生存技能，最终变成了一个空壳——**主人反而被动地依赖奴隶而存在**。

这也是当前 Vibe Coding 面临的最大陷阱：**当你把所有的思考、架构和实现都外包给 AI 时，AI 获得了对项目的绝对掌控权（知识层面），而你成了那个看不懂自己代码的“废物主人”。** 一旦报错，你束手无策。

**结论：** 我们需要一个框架，既能利用 AI 的高效，又能让我们深度参与项目的共创，保持对代码的绝对掌控。

## 5. 思维框架：RIPER-5 工作流 (30-45分钟)

为了解决“主奴困境”，我们引入了 **RIPER** 框架。这其实是从传统软件公司标准开发流（需求调研、方案设计、排期计划、编码测试、代码审查）演化而来，自然贴合工程直觉。

### R - Research (研究与许愿)
* **动作：** 给模型一个愿望（Goal），然后询问模型：“如果想实现这个，我们需要了解哪些前置知识？你可以怎么帮我？”
* **工具推荐：** 使用 `zread` (比如 `https://zread.ai/cli`) 快速将网页、API 文档转化为 Markdown 喂给模型，让它拥有最新的背景知识。

### I - Innovation (解构与创新)
* **动作：** 不要一口吃成个胖子。将大需求拆解成多个小环节，与模型探讨每个环节的实现选项。
* **工具推荐：** `AskUserQuestions Tool`。在提示词中强制要求 AI：“在开始写代码前，你必须向我提出 3-5 个澄清问题。” 让 AI 帮你挖掘你没想到的盲区。

### P - Plan (制定计划)
* **动作：** 细节敲定后，不要马上写代码！先让模型输出一份具体的实现计划（步骤 1、2、3）。
* **敲定的时机（Uncertainty Detector）：** 只有当**现状（Context）、动机（Why）、实现过程（How）和验证过程（Test）** 这四个要素都被清晰定义，没有任何不确定性时，才可以定稿。

### E - Execute (执行与开发)
* **动作：** 严格按照计划一步步执行。坚持 TDD（测试驱动开发）：让 AI 先写测试用例，再写业务代码，直到所有测试亮绿灯。
* **核心技巧：进度保存。** 如果今天做不完怎么办？不要依赖 Claude 的记忆或简单的 `compact` 命令。
    * **最佳实践：** 强制模型输出一个 `handoff.md` 文件。总结当前的进度、遗留问题和下一步计划。
    * **为什么？** 因为 `handoff.md` 是人类可读的，并且**你能改**。它比不可控的记忆压缩（compact）要稳健得多。

### R - Review (审查与对齐)
* **动作：** 阶段性完成后，把最初的 Plan 甩给模型：“对照我们的计划和设计稿，审查你刚刚写的代码，看看偏移程度有多大？” 及时纠偏，防止架构腐化。

## 6. 工具链实战与演练 (45-60分钟)

最后，我们来看看这套框架在具体开源工具中的应用。

### 1. Slidet：终端版本的 PPT
* **简介：** 一个纯终端运行的 PPT 工具，你可以直接用 Markdown 编写，然后在终端渲染出幻灯片。
* **应用：** 刚才我们讲的 RIPER 流程，就可以完美地用 Markdown 记录，并一键转为 Slidet 演示。

### 2. Uigen：前端组件的 Vibe Coding
* **简介：** 这是一个简化版的 base44 平台，专门用来对前端组件进行直觉编程和实时渲染。
* **实战演示（Live Demo）：** 我们现在用 uigen 为大家现场演示，如何用 RIPER 框架，给这个工具添加一个“前端导出”功能。（*展示解构需求、制定步骤到执行测试的流转*）

### 3. CLAUDE.md：项目的系统级记忆
* **简介：** 在初始化任何项目时，第一件事就是建立一个 `CLAUDE.md` 文件。
* **用法：** 这个文件相当于项目的“系统提示词”和“记忆库”。我们将刚才在 Research 和 Innovation 阶段与模型探索出的核心结论（比如代码规范、架构约定、常用命令）保存在这个文件里。
* **收益：** 下一次打开会话，或者换一个新的 AI 助手时，只需要让它优先读取 `CLAUDE.md`，它就能瞬间恢复所有的项目上下文，无缝衔接你的工作流。

---

**结语：** Vibe Coding 不是把方向盘交给 AI 自己去睡觉，而是成为一名清醒的领航员。掌握 `显化需求` 的沟通技巧，运用 `RIPER` 框架步步为营，你就能在这个 AI 时代，既享受极速开发的快感，又永远立于不败之地。谢谢大家！

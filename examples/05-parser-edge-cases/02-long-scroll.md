# 长内容滚动

这一页故意很长，用来观察浏览模式和全屏模式在多段文字下是否还能读。

第一段：当一页里内容明显超过终端高度时，用户会依赖 PageDown 和 PageUp，而不是只靠换页。

第二段：如果文本块切分不稳定，长文的中间就可能出现莫名其妙的空行，或者把不同段落挤在一起。

第三段：这个样例不追求视觉美观，只追求在终端里能稳定产生足够长的内容。

第四段：继续补充一些描述。真实课程讲稿会同时包含定义、例子、反例、注意事项和结论，所以“只有两三段短文”的 happy path 基本不够用。

第五段：手工 smoke 时应检查三件事，第一是滚动是否生效，第二是上下翻页后 scroll 是否复位，第三是浏览列表中的选中项是否和右侧正文一致。

第六段：到这里长度已经足够接近真实风险面了。

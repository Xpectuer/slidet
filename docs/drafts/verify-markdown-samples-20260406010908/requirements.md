# Requirements: Verification — slidet Markdown 示例输入

## 1. Problem Statement

`slidet` 已具备基础的 loader、Markdown parser、图片降级和 UI 逻辑，但当前 `slidet/examples/` 只覆盖很窄的一组输入形态。缺少系统化的 Markdown 示例会让后续验证停留在“代码能编译、几条单测能过”的层面，无法有效发现真实讲稿中的解析回归、图片路径问题、滚动阅读问题和多页目录排序问题。

## 2. Desired Outcome

构建一组可以直接被 `slidet <dir>` 消费的示例目录，覆盖基础文本、复杂排版、图片回退、超长内容和多页叙事等场景；同时定义一套轻量验证策略，确保这些样本既能支撑自动化 smoke test，也能支撑人工浏览回归。

## 3. Constraints

- **Tech stack**: Rust 2021，`pulldown-cmark`，`ratatui`，`crossterm`
- **Test frameworks**: 现有为 Rust 内嵌单测，命令入口是 `cargo test --manifest-path slidet/Cargo.toml`
- **Environments**: 本地终端优先；图片能力依赖具体终端，需允许降级文本输出

## 4. Scope

### Verification Types

- [x] Automated tests (unit / integration / e2e)
- [x] Smoke tests
- [ ] Performance tests
- [x] Manual checklists

### In Scope

- 设计并落盘多组 Markdown 示例目录
- 覆盖标题、段落、列表、引用、分隔线、代码块、图片、缺图、长文滚动、多页排序
- 为后续增加基于样例目录的 loader/parser/UI smoke test 提供输入资产
- 定义本轮验证完成标准

### Out of Scope

- 不在本轮实现完整截图对比或 golden snapshot 测试
- 不在本轮引入 CI、覆盖率平台或性能基准
- 不要求当前 parser 立刻完整支持 table、task list 等所有 Markdown 语义

## 5. Acceptance Criteria

- [ ] `slidet/examples/` 下新增至少 3 组面向回归的 Markdown 示例目录
- [ ] 新样例覆盖文本密集、图片降级、复杂结构和多页浏览四类风险
- [ ] 每组示例可被 `slidet` 目录加载器按文件名字典序稳定读取
- [ ] 示例中的图片路径既包含可解析资源，也包含故意缺失的回退场景
- [ ] 验证计划明确自动化检查、手工 smoke 路径和完成判定

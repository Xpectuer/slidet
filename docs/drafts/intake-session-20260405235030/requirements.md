# Requirements: Slide t 终端 Markdown Slide 播放器

## 1. Problem Statement
用户要在 `slidet/` 目录下创建一个新的 Rust 终端应用 `Slide t`，用于播放本地 Markdown 课件。当前仓库虽然有大量 Markdown 文档，但没有一个专门的终端 slide 播放器来把一组 Markdown 文件组织成可浏览、可播放的演示内容。

用户明确需要：
- 从文件系统读取一组 Markdown 文件作为 slide 数据源
- 在终端中同时看到 slide 列表与当前页内容
- 进入播放模式时让当前 slide 铺满整个 terminal
- 第一版先支持 Markdown 文本和图片，不追求复杂演示特性

这个需求的价值在于，把现有 Markdown 课件直接转成终端内可演示的 slide 播放体验，降低内容准备成本，并为后续演示能力扩展提供基础架构。

## 2. Desired Outcome
完成后的第一版 `Slide t` 应满足以下可观察结果：

- 在仓库 `slidet/` 下存在一个可编译运行的 Rust + `ratatui` 项目
- 用户指定一个目录后，程序能读取其中的多个 `.md` 文件，并按顺序生成 slide 列表
- 主界面采用左右两栏
- 左栏展示 slide 文件列表
- 右栏渲染当前选中的 Markdown slide
- 程序支持基础键盘导航与播放模式切换
- 播放模式下，右栏内容全屏显示，铺满 terminal
- Markdown 内容在第一版至少支持文本和图片
- 图片在支持终端图片协议时尽量真实显示；不支持时优雅降级为占位或路径提示

## 3. Constraints
- **Tech stack**:
  - 新项目目录：`slidet/`
  - 语言：Rust
  - TUI 框架：`ratatui`
  - 数据源：文件系统目录中的 Markdown 文件
- **Hard boundaries**:
  - 第一版采用“一个目录下每个 `.md` 文件对应一张 slide”的文件模型
  - 第一版只要求支持 Markdown 文本和图片渲染
  - 第一版交互只做基础键盘导航与全屏播放切换
  - 第一版以 macOS 常见 terminal 为主要目标环境，其他环境尽量兼容
  - 第一版不要求跨平台图片协议完全一致
  - 第一版不做复杂演示能力，例如计时器、讲师备注、动画步进、远程控制
  - [UNCERTAIN] 具体图片协议适配优先级尚未最终指定
  - [UNCERTAIN] 具体键位映射尚未最终指定

## 4. Scope
### In Scope
- 在 `slidet/` 创建新的 Rust `ratatui` 项目
- 扫描指定目录中的 Markdown 文件
- 按文件名顺序展示 slide 列表
- 左右两栏布局：左栏列表，右栏当前页渲染
- 基础导航：浏览列表、切换当前 slide、进入/退出播放模式
- Markdown 文本渲染
- Markdown 图片渲染
- 终端支持图片协议时显示图片；不支持时降级显示
- 播放模式下右栏内容全屏铺满 terminal

### Out of Scope
- 单个 Markdown 文件内通过分隔符拆分多页 slide
- manifest / 清单文件配置顺序或元数据
- 自动播放
- 演讲计时器
- speaker notes / 演讲者视图
- 动画步进、增量 reveal
- 复杂 Markdown 扩展能力
- 跨平台一致性专项优化
- 非 Markdown 数据源
- 视频、代码执行、交互式组件等高级内容类型

## 5. Acceptance Criteria
- [ ] `slidet/` 下存在可构建运行的 Rust `ratatui` 项目骨架
- [ ] 程序可接收一个目录作为 slide 数据源，并识别其中的 `.md` 文件
- [ ] 左栏能列出该目录中的 Markdown slides，顺序基于文件名
- [ ] 右栏能渲染当前选中的 Markdown slide 文本内容
- [ ] 用户可通过基础键盘操作切换当前 slide
- [ ] 用户可进入播放模式，并让当前 slide 内容铺满 terminal
- [ ] Markdown 中的图片在支持的 terminal 中尝试真实渲染
- [ ] 当 terminal 不支持图片协议时，程序不会崩溃，并提供可理解的降级显示
- [ ] 第一版实现不依赖 manifest、自动播放、speaker notes 等扩展特性
